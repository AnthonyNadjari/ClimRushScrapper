Comment ca marche

1

Configurez

Editez config/segments.json sur GitHub. Ajoutez des segments, des requetes de recherche, des zones geographiques.

2

Ca tourne tout seul

Le commit declenche GitHub Actions. 16 segments tournent en parallele sur des serveurs GitHub. Chaque segment scrape Google Maps + Pages Jaunes avec expansion par arrondissement.

3

Resultats

Un Excel pro est genere automatiquement et commite dans le repo. Telechargez-le depuis le Dashboard.

Architecture

🔍 Sources

  • Google Maps — scroll + clic sur chaque fiche, extrait: nom, adresse, tel, site, note, avis
  • Pages Jaunes — recherche + pagination, clic pour reveler le tel

⚡ Performance

  • 16 runners en parallele (1 par segment)
  • 5 queries simultanees par runner
  • Expansion par arrondissement : chaque requete × 20 arrondissements
  • ~15-20 min au total au lieu de 2h+

🛡️ Robustesse

  • 3 retries par requete avec delai exponentiel
  • Si un segment plante, les autres continuent
  • Deduplication par nom + code postal
  • Filtrage: contact requis (tel ou site), zone valide

📦 Output

  • Excel pro avec dashboard + 1 onglet par segment
  • CSV consolide
  • JSON pour ce dashboard web
  • Score qualite: tel(3) + site(2) + email(3) + adresse(1) + note(1)
